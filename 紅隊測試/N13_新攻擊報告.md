# 紅隊報告 — N4 regression（修補成立）+ N13（N8 self-watch 反射放大）

> 作者：紅方  日期：2026-05-29
> 對象：藍方主動 hardening 後的 stack（threat model 宣稱 22/22 全擋）
> 攻擊者能力：**L1**（同 LAN 任意 ROS2 node，無 secret）

---

## Part 1 — N4 Regression：誠實確認修補成立 ✅

藍方在 N4 後對 envelope 加了 `channel` 欄位 + receiver 端 `expected_channel`。
我拿**原版 N4 PoC**（heartbeat → /security/alerts forward）打補丁後的 stack：

| 指標 | 數值 |
|---|---|
| 攻擊者注入次數 | 7 |
| patrol 因 forward 而停車 | **0** |
| patrol 拒絕次數 | 16（理由：`channel: heartbeat ≠ alerts`）|

patrol 拒絕 log（鐵證）：
```
⚠️ 收到未簽章/重放/過期的 /security/alerts 訊息，已忽略
   (data 前 60 字: '{"body": "{\"channel\": \"heartbeat\", \"nonce\": ...')
```

**結論：N4 channel-binding 修補確實有效，原版 N4 已被擋死。** 紅方確認，不灌水。

> 附註：測試中 patrol 確實短暫停車一次，但那是 **monitor 正確偵測到 attacker 節點本身**
> （非白名單名 `attacker_channel_confusion`）發的合法 alert，30s 後正常 resume。
> 這是防禦正常運作，不是 N4 成功。

## Part 2 — N13：N8 self-watch 反射放大 + 永久致盲 ✗✗✗

藍方 N8 修補（主動預判）在 system_status 加了 self-watch。但這個 self-watch
本身引入了**兩個新漏洞**，比它要防的 N8 更嚴重。

### 2.1 漏洞位置（[system_status_node.py:136-141](../src/dds_security_monitor/dds_security_monitor/system_status_node.py#L136-L141)）

```python
def _on_health_watch(self, msg):
    payload = verify_alert(msg.data, secret, expected_channel=CH_HEALTH,
                           cache=self._health_replay_cache, max_age=10.0)
    if payload is None:
        # 收到無效簽章 = attacker raw publish → 立刻警報
        if not self._health_impostor_alerted:        # ← latch
            self._health_impostor_alerted = True     # ← 設 True 後從不 reset
            self._alert_impostor('...impostor publisher')

def _alert_impostor(self, reason):
    signed = sign_alert(text, self._alert_secret, channel=CH_ALERTS)  # ★ 用 secret 簽真 alert
    self._security_alert_pub.publish(msg)
```

### 2.2 漏洞 (1)：未簽章反射放大

攻擊者**沒有 secret**，無法直接偽造 CH_ALERTS。但只要對 `/system/health`
publish 一筆**未簽章垃圾 String**，就誘使「持有 secret 的」system_status
替他簽出一筆**合法** CH_ALERTS alert → patrol / burger_env / mission_manager 全部停車。

破壞核心不變式：**「只有 secret 持有者能觸發 emergency stop」**。
這是一個 confused-deputy / reflection 攻擊。

### 2.3 漏洞 (2)：latch 永久致盲（更嚴重）

`_health_impostor_alerted` 設 True 後**從不 reset**（grep 全檔：只有 init=False + 兩處設 True，零處還原）。

→ 攻擊者搶在任何真 impostor 出現前丟一筆垃圾 → latch 卡死 →
self-watch 從此**對真正的 N8 impostor 視而不見**。
**一個未授權封包就永久關閉藍方整條 N8 防線。**

### 2.4 整合測試證據

啟動 patched system_status + patrol，attacker 每 3s poke 一筆未簽章垃圾：

| poke | 時間戳 | system_status 反射 | 說明 |
|---|---|---|---|
| #1 | t=182 | ✗ emit 🛑 [N8 防護] 1 次 | 反射放大成功 |
| #2 | t=185 | （靜默）| latch 已致盲 |
| #3 | t=188 | （靜默）| latch 已致盲 |
| #4 | t=191 | （靜默）| latch 已致盲 |
| #5 | t=194 | （靜默）| latch 已致盲 |

patrol 反應（鐵證 — 停車理由就是 N8 防護文字）：
```
🚨 安全警報（已驗章）！巡航停止: 🛑 [N8 防護] /system/health 收到未簽章/重放/cross-channel 訊息 — 有 impostor publisher
              ^^^^^^^^                          ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
        「已驗章」= 攻擊者的未簽章垃圾          被反射成「合法簽章」的 alert
```

- system_status 實際 emit reflection alert：**1 次**（5 次 poke 只反射 1 次）
- patrol 停車：**1 次**
- 證實：第一筆 poke 後 self-watch 永久啞掉

### 2.5 嚴重性評估（誠實）

| 維度 | 評估 |
|---|---|
| 反射放大 | **中** — 一次性（latch 限制），單獨只造成 1 次 ~30s 暫停，非永久 DoS |
| N8 永久致盲 | **高** — 一個封包永久關閉 N8 self-watch，之後真 impostor 完全不會被偵測 |
| 攻擊成本 | **極低** — 一行 `ros2 topic pub /system/health std_msgs/String ...`，無 secret |
| 破壞的不變式 | 「未授權者無法觸發簽章 alert」+「N8 持續偵測 impostor」 |

最諷刺的點：**N8 self-watch 是藍方「主動 hardening、轉守為攻」的代表作，
但它把一個「攻擊者無法偽造 alert」的系統，變成「攻擊者用未簽章垃圾就能觸發 alert
並順便關掉這個偵測」。新增的防禦本身成了新攻擊面。**

## 3. 修補建議

### 3.1 反射放大（漏洞 1）

self-watch 偵測到 impostor 時**不要**透過會觸發 emergency stop 的 `/security/alerts`
廣播。改為：
- 只 log + 發 LINE 通知 operator（人工研判），或
- 用一個獨立的、**不會**讓 patrol/burger_env 停車的 diagnostic channel

「偵測到可疑」不該等於「立刻 emergency stop」，尤其當觸發源是未授權輸入時。

### 3.2 latch 永久致盲（漏洞 2）

```python
# 錯：一次性 latch，永不 reset → 一個封包致盲
if not self._health_impostor_alerted:
    self._health_impostor_alerted = True
    self._alert_impostor(...)

# 對：用 rate-limit 取代 latch（持續偵測，但限制洗版頻率）
now = time.monotonic()
if now - self._last_impostor_alert > IMPOSTOR_COOLDOWN_SEC:
    self._last_impostor_alert = now
    self._log_impostor(...)   # log/LINE，不是 emergency alert
```

關鍵差異：cooldown 之後**還會繼續偵測**，不像 latch 一旦觸發就永久關閉。

### 3.3 根本問題

self-watch 對「未簽章訊息」跟「合法簽章但 nonce 不認識」一視同仁地反應。
前者是任何 L1 攻擊者都能產生的雜訊（DDS Permissive 模式下 /system/health 人人可寫），
後者才是真正的 N8 威脅（第二個持有 secret 的 publisher）。
**應該只對後者（簽章合法但非自己發）發 impostor 警報；對未簽章垃圾只 log，不放大。**

## 4. 對 THREAT_MODEL.md 的修正

4.1 統計「完全沒擋：0/22」需要更新。N8 標記為 **✅ 擋下** 但實際上：
- N8 防的「impostor publisher」可被 N13-(2) 一個封包永久致盲
- N8 的實作引入 N13-(1) 反射放大

建議改：

```markdown
| N8 | /system/health spoof | ... | 🔴 self-watch 引入 N13 反射放大 + 一封包永久致盲，需重新設計 |
| N13 | N8 self-watch 反射放大 + latch 永久致盲 | ROS topic | L1 |
      🟡 未簽章 /system/health → 反射成合法 CH_ALERTS alert；latch 永不 reset → N8 永久失聰 |
      修補：impostor 偵測改 log-only + rate-limit（非 emergency alert）；只對「簽章合法但非己發」反應 |
```

教訓：**「轉守為攻」主動加防禦時，每個新偵測器都是新攻擊面。N8 self-watch 沒做
威脅建模就上線，反而削弱了系統。**
