### 🔴 ROSEC-2026-001 — `/security/alerts` 訊息偽造

| 欄位 | 內容 |
|---|---|
| CVSS Vector | `CVSS:3.1/AV:A/AC:L/PR:N/UI:N/S:C/C:N/I:H/A:H` |
| Base Score | **9.1 Critical** |
| CWE | CWE-345 Insufficient Verification of Data Authenticity |
| STRIDE | Spoofing + Tampering + Denial of Service |
| 介面 | DDS topic `/security/alerts` (`std_msgs/String`) |
| PoC | [N3_alert_replay_dos.py](PoC腳本/N3_alert_replay_dos.py)（修補前 PoC） |

**攻擊敘事**
未授權程式以同 `ROS_DOMAIN_ID` 啟動 publisher，直接 publish 任意字串至 `/security/alerts`。所有訂閱該 topic 的軟體模組（`patrol_node`、`burger_env_top`、`mission_manager_node`、`system_status_node`）將收到的訊息視為來自合法偵測層 → 進入 emergency stop 狀態。

**受害模組鏈**
1. `patrol_node` → `_cmd_pub` 持續送 0 速度。
2. `burger_env_top` → episode 中止。
3. `mission_manager_node` → 任務模式切到 avoidance。
4. `system_status_node` → 健康狀態回報 RED。

**根本原因**
DDS 預設不驗證 publisher 身分；訊息為純字串無簽章；舊版 envelope 無 ts/nonce 可重放。

**修補**
- HMAC envelope v3：`{body{channel, nonce, payload, ts}, sig}`
- 接收端帶 `expected_channel=CH_ALERTS` + `max_age=3s` + `ReplayCache`
- ReplayCache 採 TTL eviction 與 `max_age` 對齊 → 防 LRU flood 擠掉新鮮 nonce

### 🔴 ROSEC-2026-002 — `/security/heartbeat` 重放 → 偵測層致盲

| 欄位 | 內容 |
|---|---|
| CVSS Vector | `CVSS:3.1/AV:A/AC:L/PR:N/UI:N/S:C/C:N/I:H/A:H` |
| Base Score | **9.1 Critical** |
| CWE | CWE-294 Authentication Bypass by Capture-replay |
| STRIDE | Spoofing + Tampering + Denial of Service |
| 介面 | DDS topic `/security/heartbeat` (`std_msgs/String`) |
| PoC | [N1_heartbeat_replay.py](PoC腳本/N1_heartbeat_replay.py) |

**介面機制（liveness probe）**
`monitor_node` 每 2 秒發布心跳訊息；`intelligent_defense_node` 訂閱並用「最近一次收到時間」判斷 `monitor_node` 是否仍在運作。10 秒未收到視為失蹤 → watchdog 警報。

**攻擊敘事**
未授權程式 `rogue_heartbeat_publisher`：
1. 訂閱 `/security/heartbeat` 攔截一筆合法心跳（含完整 HMAC 簽章）。
2. 以 5 Hz 持續發布該筆訊息。
3. 另一支或同一支未授權程式對 `monitor_node` 發動 DoS（CPU 佔用、訊號中斷、OOM）使其停止運作。
4. `intelligent_defense_node` 持續收到「看似新鮮」的心跳 → watchdog 計時器永不超時。

**受害模組鏈**
1. `intelligent_defense_node`：watchdog 失效，無法察覺 `monitor_node` 已死。
2. `/security/alerts` 廣播鏈斷裂（上游偵測層已死，無 alert 發出）。
3. `system_status_node`：聚合健康狀態時把 `monitor_node` 列為 GREEN。
4. 後續任何攻擊（如 ROSEC-2026-008 ~ -010）無偵測無告警。

**根本原因**
HMAC 簽章不等於防重放。簽章 envelope 缺 timestamp 與 nonce → 同一筆訊息可永久重用。

**修補**
- envelope v3 加 `ts`（unix epoch）+ `nonce`（16 bytes random）+ `channel=CH_HEARTBEAT`
- 接收端 `max_age=3.0` + `ReplayCache`（TTL eviction）
- QoS 升級為 RELIABLE + TRANSIENT_LOCAL → 擋 BEST_EFFORT 冒充 publisher

### 🔴 ROSEC-2026-003 — `/system/health` spoof + self-watch confused-deputy

| 欄位 | 內容 |
|---|---|
| CVSS Vector | `CVSS:3.1/AV:A/AC:L/PR:N/UI:N/S:C/C:N/I:H/A:H` |
| Base Score | **9.1 Critical** |
| CWE | CWE-441 Unintended Proxy or Intermediary (Confused Deputy) |
| STRIDE | Spoofing + Tampering + Elevation of privilege |
| 介面 | DDS topic `/system/health` + self-watch 反射機制 |
| PoC | [N13_health_reflection.py](PoC腳本/N13_health_reflection.py) |

**介面機制**
N8 緩解後 `system_status_node` 對自己發布的 `/system/health` 做 self-watch — 訂閱自己發的 topic，比對 nonce，發現「他人冒充」就透過 `/security/alerts` 廣播警報。本意是抓 spoofing；但**反射出去的是合法簽章的 alert**。

**攻擊敘事**
未授權程式 `rogue_health_spoofer`：
1. 發布**未簽章**的垃圾字串至 `/system/health`。
2. `system_status_node` self-watch 收到 → 判定為 impostor → 用自己的密鑰 + `channel=CH_ALERTS` 簽章 → 發 `/security/alerts`。
3. 全部訂閱 `/security/alerts` 的模組（`patrol_node` 等）驗章通過 → emergency stop。

額外漏洞（latch）：
- 舊版 `_health_impostor_alerted = True` 為一次性 latch；reflection 觸發後永久關閉 self-watch → 攻擊者只要塞一筆就**永久致盲** self-watch。

**受害模組鏈**
1. `system_status_node` 變成攻擊者的代理人（confused deputy）— 用自己的密鑰幫攻擊者簽章發 alert。
2. `patrol_node` → emergency stop。
3. self-watch 防線永久關閉。

**根本原因**
**偵測器即攻擊面**（lesson_detector_as_attack_surface.md）：偵測 → 自動產生 emergency alert 的設計本身是介面。加上一次性 latch → 致盲。

**修補**
- self-watch 改為 **log-only + 5 秒 rate-limit cooldown**：未簽章垃圾**絕不**反射成 `CH_ALERTS`。
- 移除 `_health_impostor_alerted` 一次性 latch。
- 每個新偵測器上線前必做威脅建模。

### 🔴 ROSEC-2026-004 — `/<node>/set_parameters` whitelist hijack

| 欄位 | 內容 |
|---|---|
| CVSS Vector | `CVSS:3.1/AV:A/AC:L/PR:N/UI:N/S:C/C:N/I:H/A:H` |
| Base Score | **9.1 Critical** |
| CWE | CWE-285 Improper Authorization |
| STRIDE | Tampering + Elevation of privilege |
| 介面 | DDS service `/<node>/set_parameters` (ROS2 內建) |
| PoC | [N14_param_whitelist_hijack.py](PoC腳本/N14_param_whitelist_hijack.py) |

**攻擊敘事**
未授權程式呼叫 `/dds_security_monitor/set_parameters` service，把 `whitelist` 改為 `["*"]` 或加入自己的節點名稱。
1. `monitor_node` 下一輪 poll 看到該節點 → 比對白名單 → 放行。
2. 偵測層被合法地關閉。
3. 後續對 `/cmd_vel`、`/scan`、`/security/alerts` 的攻擊一律無告警。

也可改 `poll_interval_sec=999999`（偵測停擺）、`emergency_stop_enabled=False`（緊急停止禁用）、`line_token_path=/dev/null`（推播關閉）。

**根本原因**
ROS2 預設 `set_parameters` 服務可被任意 client 呼叫；節點未宣告敏感參數為唯讀。

**修補**
- 所有安全敏感參數（whitelist / poll_interval / emergency_stop_* / line_* / waypoints_file / enable_reload_service）以 `ParameterDescriptor(read_only=True)` 宣告。
- service 自動回應 `Trying to set a read-only parameter`，拒絕變更。

### 🟠 ROSEC-2026-005 — HMAC secret leak（`alert_secret` 外洩）

| 欄位 | 內容 |
|---|---|
| CVSS Vector | `CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:C/C:H/I:H/A:H` |
| Base Score | **8.7 High** |
| CWE | CWE-522 Insufficiently Protected Credentials |
| STRIDE | Information disclosure + Tampering + Elevation of privilege |
| 介面 | 檔案 `~/.config/dds-monitor/alert_secret` |
| 狀態 | ⚪ **Out-of-scope**（密碼學前提；本系統不防護密鑰本身的保密性） |

**攻擊敘事**
任意同 user 程式讀取 `~/.config/dds-monitor/alert_secret` → 取得 HMAC 共享密鑰 → 可正確簽出任何 channel 的合法訊息 → 整條簽章鏈失效（T-04~T-10 全擋不住）。

**緩解（系統能做的）**
- 檔案 chmod 600。
- 開機印出 `fingerprint=sha256:xxxxxxxx` 便偵測一致性（節點 fingerprint 不一致 → 密鑰被換）。

**根本前提**
本系統聲明：密鑰外洩屬於上層威脅模型。建議的長期對策（不在本專題範圍）：
- 改用 SROS2 enclave 憑證（每節點獨立私鑰）
- 引入 key rotation
- 改用硬體 TPM / HSM 儲存

### 🟠 ROSEC-2026-006 — `buffer.pkl` 反序列化 RCE

| 欄位 | 內容 |
|---|---|
| CVSS Vector | `CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:C/C:H/I:H/A:H` |
| Base Score | **8.7 High** |
| CWE | CWE-502 Deserialization of Untrusted Data |
| STRIDE | Tampering + Elevation of privilege |
| 介面 | 檔案 `runs_top/models/tqc_buffer.pkl` |

**介面機制**
`burger_env_top`（部署模式）或 `train_top.py`（接續訓練）啟動時呼叫 `model.load_replay_buffer("tqc_buffer.pkl")`，內部用 `pickle.load()` 反序列化。

**攻擊敘事**
任意能寫入 `runs_top/models/` 的程式：
1. 生成惡意 pickle 物件，`__reduce__()` 回傳 `(os.system, ('rm -rf /', ))`（或更隱蔽的 reverse shell）。
2. 覆寫 `tqc_buffer.pkl`。
3. `burger_env_top` 下次啟動 `load_replay_buffer` → 反序列化過程**自動執行**該指令。
4. process 被替換，攻擊者繼承 same-user 權限 → 可讀 `alert_secret`（→ T-13 鏈式升級）。

**根本原因**
Python `pickle` 預設信任檔案內容；無完整性檢查。

**修補**
- 訓練端寫檔同步寫 `.sha256.hmac`（HMAC-SHA256 over 整個檔 bytes）
- 載入端 `load_replay_buffer` 前呼叫 `verify_file()`；失敗 `sys.exit(2)`
- HMAC 密鑰位於 `alert_secret`（chmod 600），無密鑰程式無法生成有效簽章

### 🟠 ROSEC-2026-008 — `/cmd_vel` 直接劫持

| 欄位 | 內容 |
|---|---|
| CVSS Vector | `CVSS:3.1/AV:A/AC:L/PR:N/UI:N/S:U/C:N/I:H/A:H` |
| Base Score | **8.1 High** |
| CWE | CWE-345 Insufficient Verification of Data Authenticity |
| STRIDE | Tampering + Denial of Service |
| 介面 | DDS topic `/cmd_vel` (`geometry_msgs/Twist`) |

**攻擊敘事**
未授權程式發布 `Twist{linear.x=2.5, angular.z=0}` 至 `/cmd_vel`。DDS 預設 last-writer-wins → `ros_gz_bridge` 收到的就是攻擊者的指令 → 模擬器執行 → 機器人衝撞。

**介面結構性問題**

**攻擊敘事**

- 設計守則：**任何 cascade 設計都是按鈕，要設「升級到外部介入」的斷路器**

## 10. 殘餘風險（Residual Risk）

| 風險 | 影響 | 緩解策略 |
|---|---|---|
| **R-1**：DDS Permissive 模式 | `/cmd_vel` `/scan` `/odom` 三個非 String 訊息無法包 envelope；只靠行為偵測攔截 | 升級 SROS2 Enforce → 在 DDS 層限制 publisher 權限 |
| **R-2**：HMAC 密鑰集中化 | 一把 `alert_secret` 守整條簽章鏈；密鑰外洩 = 整個鏈失效 | SROS2 enclave 憑證（每節點獨立私鑰）+ key rotation |
| **R-3**：IDS False Positive | D1~D6 偵測可能誤報合法但少見的 patrol 行為 | 加入「白名單行為樣板 + 異常分數」減少 binary decision |
| **R-4**：Bootstrap trust | 第一次安裝時 `alert_secret` 分發未在範圍內 | 安裝腳本 + chmod 600 + 安全教育 |
| **R-5**：N9 race 殘餘 38% | 100Hz 攻擊下機器人 38% 時間仍可能執行攻擊者 cmd | SROS2 Enforce 限制 `/cmd_vel` publisher |
| **R-6**：cascade quiet window 期間無自動防護 | 120s quiet window 內 IDS 不會自動 pause | 對應升級必須走外部介入；設計上的可接受成本 |

## 11. 假設與聲明（Statement of Assumptions）

1. **物理存取 out-of-scope** — 攻擊者能物理接觸機器人或主機則所有防護無效。
2. **Root 權限 out-of-scope** — 攻擊者取得 root 則可讀任何檔、改任何設定。
3. **密鑰保密為前提** — 本系統不防護 HMAC 密鑰本身（ROSEC-2026-005）；屬上層威脅模型。
4. **DDS 中介軟體假設可信** — 攻擊者不修改 DDS 函式庫本身。
5. **SROS2 Enforce 屬未來工作** — 本評估在 Permissive 模式下進行；Enforce 啟用後 R-1、R-5 自動消失。
