# ROS2 DDS 安全監控系統 — 檔案結構說明

## 系統架構概覽

```
三大模組
├── 模組 A：強化學習自主導航 (SAC)           ← 機器人 AI 控制
├── 模組 B：DDS 資安監控 (SROS2 + Monitor)   ← 節點防護 + 入侵偵測
└── 模組 C：任務管理與系統狀態               ← 整合協調層
    └── 附加工具：LLM 模糊測試系統 (~/llm_fuzzer)  ← 驗證資安防禦強度
```

---

## 模組 A：SAC 深度強化學習導航

### 核心定位

使用 **Soft Actor-Critic（SAC）** 讓機器人從零學習在障礙物環境中自主導航（連續動作空間）。即使切換為 AI 控制，DDS Security 仍能偵測未授權節點，觸發緊急停止——證明資安層與導航層解耦。

### 為什麼選 SAC（不用 DQN）

| 特性 | DQN（舊）| SAC（現在）|
|------|---------|-----------|
| 動作空間 | 離散（5 個固定動作）| 連續（任意速度 / 方向）|
| 探索方式 | 手動調 ε，容易卡局部最優 | 最大熵自動調，天生防轉圈 |
| 移動平滑度 | 跳躍感 | 接近人類駕駛 |

### 訓練規格

| 項目 | 說明 |
|------|------|
| 演算法 | Soft Actor-Critic (SAC) — Stable Baselines3 v2.8 |
| Observation | 12 維：LiDAR 10扇區 + 目標距離 + 目標方位角 |
| Action | 連續 [v, ω]，映射到 [0, 0.25] m/s 和 [-1.5, 1.5] rad/s |
| Reward | 靠近目標 +2.0 / 到達 +10.0 / 碰撞 −10.0 / 時間懲罰 −0.1 |
| 目標 | 3,000,000 timesteps，每 50,000 steps 存 checkpoint |
| GPU | NVIDIA RTX 5070 Laptop（CUDA，SB3 自動使用）|

### 訓練相關檔案

| 檔案 | 說明 |
|------|------|
| `src/turtlebot3_dqn/turtlebot3_dqn/burger_env.py` | SAC Gym 環境、Domain Randomization、完整 Reward |
| `src/turtlebot3_dqn/turtlebot3_dqn/train_sac.py` | SAC 訓練主程式、TensorBoard、checkpoint |
| `src/turtlebot3_dqn/turtlebot3_dqn/models_sac/sac_burger_latest.zip` | 最新 SAC 模型 |
| `src/turtlebot3_dqn/turtlebot3_dqn/models_sac/checkpoints/` | 定期 checkpoint（每 50,000 steps）|
| `src/turtlebot3_dqn/turtlebot3_dqn/logs_sac/tensorboard/` | TensorBoard log |

## 模組 B：DDS 資安監控

### SROS2 三層防護架構

| 防護層 | 技術 | 效果 |
|--------|------|------|
| 身份驗證 | X.509 PKI 憑證 | 沒有憑證的節點無法加入 DDS Domain |
| 加密傳輸 | AES-256-GCM | 竊聽封包看不到任何訊息內容 |
| 存取控制 | permissions.xml 最小權限 | 有憑證但無授權也無法 pub/sub |

### SROS2 設定檔

| 檔案 | 說明 |
|------|------|
| `~/ros2_security_keystore/enclaves/governance.xml` | DDS 安全政策：禁止匿名、AES-256 加密、寫入存取控制 |
| `~/ros2_security_keystore/enclaves/*/permissions.xml` | 節點存取控制：最小權限，每節點各一份（共 9 個）|
| `~/ros2_security_keystore/enclaves/*/cert.pem` | 節點身份憑證：X.509，有效期至 2036 年 |

### OMG DDS Security 1.1 規範對應

| 規範插件 | 實作方式 | 狀態 |
|---------|---------|------|
| Authentication Plugin | X.509 PKI 憑證身份驗證 | ✅ |
| Access Control Plugin | permissions.xml 最小權限存取控制 | ✅ |
| Cryptography Plugin | AES-256-GCM 加密 + HMAC 完整性驗證 | ✅ |
| Logging Plugin | 安全事件日誌 | ❌ |
| Data Tagging Plugin | 資料機密等級標記 | ❌ |

### 安全監控節點

| 節點 | 訂閱 | 發布 | 說明 |
|------|------|------|------|
| `monitor_node.py` | — | `/security/alerts` | 掃描節點圖，發現未知節點觸發緊急停止 |
| `patrol_node.py` | `/scan` `/security/alerts` | `/cmd_vel` | LiDAR 障礙物迴避，收到警報立刻停止 |
| `sensor_hub_node.py` | `/scan` `/imu` | `/sensor/status` | 彙整 LiDAR + IMU |
| `mission_manager_node.py` | `/sensor/status` `/security/alerts` | `/mission/cmd` | 依感測器狀態切換任務模式 |
| `system_status_node.py` | `/sensor/status` `/mission/cmd` `/security/alerts` | `/system/health` | 每 2 秒發布系統健康報告 |

所有節點路徑：`src/dds_security_monitor/dds_security_monitor/`

## 附加工具：LLM 模糊測試系統

位置：`~/llm_fuzzer/`（獨立專案，不在 ros2_ws 內）

### 用途

驗證資安防禦對惡意 JSON payload 的防護強度。透過 GPT-4o-mini 自動生成破壞性測資，在 Docker 沙箱執行，GPT-4o 分析 ASan 崩潰報告。

### 架構

```
MutationAgent (GPT-4o-mini)
    → asyncio.Queue
    → 4x Consumer → Docker 沙箱 (ASan)
    → TriageAgent (GPT-4o) → Markdown 崩潰報告
    → SQLite 持久化 (fuzzer_state.db)
```

### 主要檔案

| 檔案 | 說明 |
|------|------|
| `~/llm_fuzzer/orchestrator.py` | 主調度器：1 Producer + 4 Consumer 非同步架構 |
| `~/llm_fuzzer/agents.py` | MutationAgent（生成測資）+ TriageAgent（分析崩潰）|
| `~/llm_fuzzer/sandbox.py` | Docker 沙箱執行器（隔離、記憶體限制、ASan）|
| `~/llm_fuzzer/models.py` | Pydantic 資料模型 |
| `~/llm_fuzzer/report.py` | 崩潰報告檢視工具 |
| `~/llm_fuzzer/Dockerfile.target` | 靶機映像：含三個刻意漏洞的 C++ 程式 |
| `~/llm_fuzzer/target.cpp` | heap-overflow / stack-overflow / integer-overflow |
| `~/llm_fuzzer/fuzzer_state.db` | SQLite：已執行 hash、崩潰報告、session 快照 |
