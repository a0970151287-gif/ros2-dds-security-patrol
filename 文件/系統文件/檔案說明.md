# ROS2 DDS 安全監控系統 — 檔案結構說明

> **版本：TQC + HMAC envelope v3 + 行為 IDS。** 本文件已從舊版（SAC / DQN / 「Enforce 假設」）更新到現況。
> 系統概述：ROS2 機器人巡邏 + 應用層資安防護 + TQC 強化學習。

## 系統架構概覽

```
三大模組
├── 模組 A：強化學習自主導航 (TQC)                       ← 機器人 AI 控制
├── 模組 B：DDS 資安監控 (SROS2 + Monitor + 行為 IDS)    ← 節點防護 + 入侵偵測
└── 模組 C：任務管理與系統狀態                           ← 整合協調層
    └── 附加工具：LLM 模糊測試系統 (~/llm_fuzzer)        ← 驗證資安防禦強度
```

## 模組 A：TQC 深度強化學習導航

### 核心定位

使用 **Truncated Quantile Critics（TQC，sb3-contrib）** 讓機器人從零學習在障礙物環境中自主導航（連續動作空間）。即使切換為 AI 控制，DDS Security 仍能偵測未授權節點與異常行為並觸發緊急停止——證明資安層與導航層解耦。

### 為什麼用 TQC（演進：DQN → SAC → TQC）

| 特性 | DQN（最舊）| SAC | TQC（現在）|
|------|-----------|-----|-----------|
| 動作空間 | 離散（5 個固定動作）| 連續 | 連續 |
| 探索方式 | 手動調 ε，易卡局部最優 | 最大熵自動調 | 最大熵 + 分位數 critic |
| Q 值估計 | 單一 Q，易高估 | double Q | 截斷分位數 critic，抑制過度樂觀高估 |
| 移動平滑度 | 跳躍感 | 接近人類駕駛 | 接近人類、更穩定 |

### 訓練規格（地面真相取自 `burger_env_top.py` / `train_top.py`）

| 項目 | 說明 |
|------|------|
| 演算法 | Truncated Quantile Critics (TQC) — sb3-contrib |
| Observation | **744 維** = (180 beam LiDAR + 6 維狀態) × frame stack K=4 |
| 特徵抽取 | 1D-Conv LiDAR encoder（`feature_extractors.py`，features_dim=256）|
| Action | 連續 `Box[-1,1]²`，映射到 lin ∈ [0, 0.22] m/s、ang ∈ [-1.5, 1.5] rad/s |
| Reward | **Potential-based shaping（Ng-Harada-Russell 1999）**：`r = γ·Φ(s′) − Φ(s)`，Φ = −目標距離；外加平滑懲罰 + waypoint 稀疏獎勵 |
| 課程學習 | Curriculum stage 1 → 5（巡邏點由 1 個漸增到全部）|
| Domain Randomization | 每 episode 隨機化 max_lin (0.18–0.22)、max_ang (1.2–1.8)；含 5% 對抗訓練 |
| 目標步數 | `TOTAL_STEPS = 2,000,000`（預期 1.0–1.5M 收斂）|
| GPU | NVIDIA RTX 5070 Laptop（CUDA，SB3 自動使用）|

## 模組 B：DDS 資安監控

### 主防線：應用層簽章 + 行為 IDS（真正「強制」生效的一層）

本系統 SROS2 採 **Permissive 模式**，DDS 層**不強制阻擋**未授權的 pub/sub（只記錄、不擋）。因此真正執行強制防護的是**應用層**：

| 防護層 | 技術 | 效果 |
|--------|------|------|
| 訊息真實性 | **HMAC envelope v3**（channel + ts + nonce + payload + sig，五道驗章）| 偽造 / 竄改 / 跨頻道混淆的 String 警報被接收端拒絕 |
| 重放防護 | ReplayCache（nonce LRU + TTL）+ 時效檢查 | 錄製的合法封包無法重放 |
| 行為入侵偵測 | 行為 IDS **D1–D6**（物理約束、scan std、cmd-vs-odom 一致性、心跳看門狗…）| 抓出 SROS2 包不到的 `/cmd_vel` `/scan` `/odom` 非 String 攻擊 |
| 節點圖監控 | `monitor_node` 掃 ROS graph 比對白名單 | 未授權節點上線即告警 |

> ⚠️ 介紹時請誠實說：「**DDS 層是 Permissive，不會擋發布**；強制防線在應用層 HMAC + 行為 IDS。要 DDS 層強制擋需切 Enforce 模式，這是未來工作。」
> **不要說「沒有憑證就無法加入 / 無法發布」——那是 Enforce 模式才成立，本系統目前是 Permissive。**

### SROS2 設定檔（已建置，目前以 Permissive 運行）

| 檔案 | 說明 |
|------|------|

| `~/ros2_security_keystore/enclaves/*/permissions.xml` | 各節點最小權限定義（每節點一份）|

### 安全監控節點（6 個資安節點）

| 節點 | 訂閱 | 發布 | 說明 |
|------|------|------|------|
| `monitor_node.py` | ROS graph | `/security/alerts`（envelope v3）、`/security/heartbeat` | 掃描節點圖、簽章告警、ReplayCache |
| `intelligent_defense_node.py` | `/scan` `/odom` `/cmd_vel` `/security/heartbeat` | `/security/alerts` | 行為 IDS D1–D6 + monitor 心跳看門狗 |
| `patrol_node.py` | `/scan` `/security/alerts` | `/cmd_vel` | LiDAR 避障；收到驗章通過的警報 → 停止 + cascade quiet window |
| `sensor_hub_node.py` | `/scan` `/imu` | `/sensor/status` | 彙整 LiDAR + IMU |
| `mission_manager_node.py` | `/sensor/status` `/security/alerts` | `/mission/cmd` | 依感測器狀態切換任務模式（驗章後動作）|
| `system_status_node.py` | `/sensor/status` `/mission/cmd` `/security/alerts` | `/system/health` | 系統健康聚合 + self-watch |

## 附加工具：LLM 模糊測試系統

位置：`~/llm_fuzzer/`（獨立專案，不在 ros2_ws 內）。**定位：輔助驗證工具；資安主力是紅隊 PoC（N1–N24）。**

### 用途

驗證資安防禦對惡意 JSON payload 的防護強度。透過 GPT-4o-mini 自動生成破壞性測資，在 Docker 沙箱執行，GPT-4o 分析 ASan 崩潰報告。

### 架構

```
MutationAgent (GPT-4o-mini)
    → asyncio.Queue
    → 4x Consumer → Docker 沙箱 (ASan)
    → TriageAgent (GPT-4o) → Markdown 崩潰報告
    → SQLite 持久化 (fuzzer_state.db)
```

### 主要檔案

| 檔案 | 說明 |
|------|------|
| `~/llm_fuzzer/orchestrator.py` | 主調度器：1 Producer + 4 Consumer 非同步架構 |
| `~/llm_fuzzer/agents.py` | MutationAgent（生成測資）+ TriageAgent（分析崩潰）|
| `~/llm_fuzzer/sandbox.py` | Docker 沙箱執行器（隔離、記憶體限制、ASan）|
| `~/llm_fuzzer/models.py` | Pydantic 資料模型 |
| `~/llm_fuzzer/report.py` | 崩潰報告檢視工具 |
| `~/llm_fuzzer/Dockerfile.target` | 靶機映像：含三個刻意漏洞的 C++ 程式 |
| `~/llm_fuzzer/target.cpp` | heap-overflow / stack-overflow / integer-overflow |
| `~/llm_fuzzer/fuzzer_state.db` | SQLite：已執行 hash、崩潰報告、session 快照 |
