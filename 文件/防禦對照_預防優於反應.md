# 防禦對照：預防（SROS2 Enforce）優於反應（行為 IDS）

**主題**：同一個 `/cmd_vel` 注入攻擊，在「無安全 + 行為偵測」與「SROS2 Enforce」兩種架構下，對機器人的實際衝擊對照。
**核心論點**：**偵測 ≠ 防護**。能在攻擊「到達機器人之前」就攔截（預防），優於「機器人已開始反應後」才偵測並急停（反應）。

---

## 一、兩層防禦的本質差異

| | 反應式（行為 IDS / Zeek） | 預防式（SROS2 Enforce） |
|---|---|---|
| 何時生效 | 攻擊**已送達**機器人、已開始執行**之後** | 攻擊在**認證層**就被擋，**從未到達**機器人 |
| 機制 | 觀察到異常行為（超物理速度、cmd-odom 不符）→ 投票 → 緊急停 | 無 CA 憑證 → 連 DDS participant 都建不起來 |
| 機器人狀態 | 已偏移／逼近障礙 → 急停 → 復歸（大費周章） | 完全無感，照常巡邏 |
| 守備對象 | **內鬼**：持有合法憑證但行為異常的節點 | **外部未授權**：沒有憑證的攻擊者 |

兩者**互補、缺一不可**——但對「外部攻擊者注入」這個情境，**預防才是正解**。

---

## 二、反應式實測（Permissive + 行為 IDS）— 機器人「大費周章」

攻擊機（10.10.10.1）對 `/cmd_vel` 注入超物理指令（linear=0.5 m/s，超過 Burger 上限 0.22）。

**實測證據**：
- **封包層**：Zeek 偵測 `[參數竄改 F1] 來源 10.10.10.1 → 14918`、`[注入]`；機器人 `/odom` 實際角速度從 baseline 0 飆到攻擊指定的 **1.8 rad/s**（機器人物理上聽攻擊者的 → 劫持成立）。
- **行為層**：`intelligent_defense_node` 投票 **2/6 觸發告警**：
  - `D1 [cmd_lin=0.50 >= 0.23]`（超物理速度）
  - `D6 [cmd v=0.10m/s 持續正向但 odom 靜止]`（cmd-vs-odom 不一致）
  - → 緊急停車。
- **後果**：此時機器人**已逼近障礙物 0.13m（⚠️ 危險）、進入 AVOID_OBSTACLE**，靠 IDS 事後急停才沒撞上。

> 問題核心：IDS 必須**先觀察到**異常指令才能判斷 → 那個壞指令**早已送達機器人、它已開始反應**。偵測救得回來，但機器人已經歪了、逼近障礙、急停、復歸——「大費周章」。

---

## 三、預防式（SROS2 Enforce）— 攻擊在門口就死，機器人無感

同一個攻擊，系統改在 SROS2 Enforce 下運行（`01c_啟動系統_enforce.sh`）。

**機制（已驗證）**：攻擊機沒有本 CA 簽的憑證 → Enforce 下 participant 建立失敗：
```
error creating node: couldn't find all security files!
```
→ 那包 `linear=0.5` 的偽造 `/cmd_vel` **根本進不了網路**、送不到 `ros_gz_bridge` → 機器人從頭到尾沒看到壞指令。

**最小驗證（test 12）**：Enforce 下攻擊者注入 `/cmd_vel`，合法訂閱端收到 **0 筆**惡意指令 → 壞指令在到達機器人前就被攔截。

**預期最終證據（待全系統 after 實測補上）**：
- 攻擊機端：`couldn't find security files`、注入器報錯、訂不到 `/odom`（被鎖在門外）。
- 目標機端：`/odom` 角速度維持 **≈ 0**（對照 before 的 1.8）→ 機器人不受影響、照常巡邏，**無急停、無逼近障礙**。

---

## 四、對照總表

| 指標 | Permissive（反應式） | Enforce（預防式） |
|---|---|---|
| 偽造 cmd 是否送達機器人 | ✅ 送達（208B → bridge 埠 14915） | ❌ 配不上，**送不到** |
| 機器人 `/odom` 角速度 | **1.8 rad/s**（=攻擊值，被劫持） | **≈ 0**（無感） |
| 機器人行為 | 逼近障礙 0.13m → AVOID → 急停 → 復歸 | 照常巡邏 |
| 防禦時機 | 攻擊**之後**偵測+急停 | 攻擊**之前**攔截 |
| 攻擊者能否觀測系統 | 能（可訂閱 /odom 等） | 不能（鎖在門外） |

---

## 五、結論

- **SROS2 Enforce = 預防**：把外部未授權攻擊擋在認證層，機器人完全不用反應。這是對「外部注入」的正解。
- **行為 IDS（D1–D6）= 反應 / 縱深最後一道**：留給「持合法憑證的內鬼行為異常」場景，不可省、但不該是第一道。
- **一句話**：**能在碰到之前攔下，就不要等碰到了再救。** 預防（Enforce）優於反應（IDS）——這正是本專案 before/after 對照要證明的核心。
