# 強化學習巡邏任務 — 計分板設計

> 把巡邏任務當作一款遊戲來設計：每個決定都有分數，Agent 的目標是讓總分最高。
> 「不同的點價值都不一樣，故障的機率會不同，你就要設計這個場景。」
>
> ⚠️ **本文件是「概念設計」**（早期構想，含優先級加權等理想化設計）。
> **實際 TQC 實作的 reward** 改用 **potential-based shaping（Ng-Harada-Russell 1999）**：
> `r = γ·Φ(s') − Φ(s) + smooth + sparse`，其中 Φ(s) = −到目標距離。
> 這個形式**理論上保證最佳策略不變**，取代下方早期的「進度 ×10」hack。
> 評估指標見 TensorBoard `scoreboard/*`（SPL / success_rate / collision_rate / curriculum_stage）。

---

## 情境設定：工廠機器人巡檢

假設場景：一台巡邏機器人在工廠廠房巡視，不同區域的設備有不同的故障率。
機器人必須**定期到達各巡邏點**，同時**避開障礙物**，並**保持通訊安全**。

---

## 巡邏點規劃（地圖分區）

| 巡邏點 ID | 位置（x, y） | 區域名稱 | 故障率 | 巡邏優先級 | 到達獎勵 |
|----------|------------|---------|-------|----------|---------|
| P1 | (-1.5, -1.5) | 電源控制室（柱子旁） | 高（30%/天）| ★★★ 緊急 | **+80 分** |
| P2 | (1.5, -1.5) | 冷卻水塔 | 中（15%/天）| ★★ 重要 | **+50 分** |
| P3 | (-1.5,  1.5) | 生產線 A | 中（12%/天）| ★★ 重要 | **+50 分** |
| P4 | (1.5,  1.5) | 生產線 B | 低（5%/天）| ★ 一般 | **+30 分** |
| P5 | (0.0, -1.8) | 出入口 | 低（3%/天）| ★ 一般 | **+20 分** |

> **設計邏輯**：故障率越高的設備，越需要被頻繁巡視，因此到達獎勵越高。

## 完整計分規則表

### 正向獎勵（得分）

| 事件 | 條件 | 分數 | 設計理由 |
|------|------|------|---------|
| 到達高優先巡邏點 | 距離 P1 < 0.25m | **+80** | 最重要的設備需優先巡視 |
| 到達中優先巡邏點 | 距離 P2/P3 < 0.25m | **+50** | 中等重要設備 |
| 到達低優先巡邏點 | 距離 P4/P5 < 0.25m | **+20~30** | 一般巡視 |
| 朝目標前進 | 每步：(上一步距離 - 現在距離) × 10 | **+0~5/步** | 鼓勵持續移動而不是原地等待 |
| 完成一輪完整巡邏 | 在 N 步內完成所有巡邏點 | **+200** | 鼓勵高效率完成任務 |
| 提前完成（速度加成） | 步數少於平均的 80%  | **+50** | 鼓勵快速高效 |

### 負向懲罰（扣分）

| 事件 | 條件 | 分數 | 設計理由 |
|------|------|------|---------|
| 碰撞障礙物 | 距離 < 0.18m | **-100，終止** | 機器人損壞代價最高 |
| 進入危險距離 | 距離 0.18m~0.35m | **-3 ~ 0/步**（漸進）| 靠得越近扣越多 |
| 不必要的原地轉圈 | 30 步內位移 < 0.2m | **-2/步** | 轉圈浪費電量和時間 |
| 角速度急跳（晃動） | 角速度變化 > 0.5 rad/s | **-0.4 × 跳變量** | 機器人應平滑移動 |
| 長時間未到達任何巡邏點 | 超過 200 步未到達任何點 | **-30** | 避免無目的遊蕩 |
| 收到資安警報時繼續移動 | 收到 /security/alerts 後仍發布 /cmd_vel | **-50** | 安全事件應優先停止 |

## 一集（Episode）的生命週期

```
開始
  │
  ▼
機器人傳送到隨機起點
  │
  ▼
目標：依序（或自主選擇）完成所有巡邏點
  │
  ├─ 到達巡邏點 → 得分 → 選下一個目標
  ├─ 碰撞障礙物 → -100，本集結束
  ├─ 超過 MAX_STEPS（600步）→ 本集結束
  └─ 完成所有巡邏點 → +200 完成獎勵，本集結束
```

---

## 目標函數（Reward Function）

```
每步 reward = 進度獎勵 + 到達獎勵 + 懲罰項

= progress_reward          # 靠近目標的距離變化 × 10
+ arrival_bonus            # 到達巡邏點時一次性獎勵（依優先級）
+ safety_penalty           # 靠牆懲罰（0 ~ -3）
+ smooth_penalty           # 角速度跳變懲罰
+ spin_penalty             # 不必要轉圈懲罰
+ security_penalty         # 收到警報後繼續移動的懲罰
+ [collision → -100, 終止]
+ [all_points_done → +200]
```

## 巡邏策略選項

機器人可以自己學習選擇巡邏順序，或者我們預設一個策略：

**選項 A：固定順序巡邏**（P1 → P2 → P3 → P4 → P5 → 重複）
- 優點：易於評估效果，行為可預測
- 缺點：沒有利用優先級動態調整

**選項 B：優先級導向（目前 TQC 設計）**
- 每次到達一個巡邏點後，選最高未訪問優先級的點
- 讓 AI 自己決定哪個最值得去（加分最高）

**選項 C（進階）：動態故障率模擬**
- 隨機觸發某個設備「故障」，故障期間該巡邏點獎勵 ×3
- 機器人要學習在故障設備附近多巡邏

---

## 與 DDS Security 的整合點

當 `monitor_node` 發出 `/security/alerts`：
1. **巡邏機器人立刻停止**（patrol_node 或 TQC agent 驗章後停止）
2. **計分板紀錄事件**：本集加上「安全事件已處理 +10」或「未正確停止 -50」
3. **人工確認後恢復巡邏**（30 秒後自動恢復，或等人工指令）

這樣把資安事件也納入 RL 的學習目標，讓機器人同時學會「高效巡邏」和「安全合規」兩件事。
