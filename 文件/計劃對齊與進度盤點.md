# 智慧工廠 ROS2/DDS 資安攻防 — 核心定義與範圍盤點

> **用途**：給組員 / 教授對齊「核心是什麼、什麼該做到最深、什麼進未來工作」。
> **決策依據**：計劃書「以**資安攻防**為主軸」＋ 教授「**專注 DDS**」＋ 原則「**深度 > 廣度**」。

---

## 0. 一句話核心（定稿）

> **「ROS2/DDS 智慧工廠的資安攻防 —— 紅隊發動三類攻擊（偵察 / 偽造注入 / DoS），藍隊在 DDS 層偵測 → 告警 → 緊急停止，並用偵測率 / 誤報率驗證有效性。」**

**篩子**：每個功能都要回答 ——「它有沒有讓『**DDS 層攻防**』更強？」沒有的，砍掉或進未來工作。

---

## 1. 三類分類

| 類別 | 功能 | 說明 |
|---|---|---|
| 🟥 **核心**（做到最深） | ① 紅隊三類攻擊（偵察 / 偽造注入 / DoS）<br>② **Zeek 在 DDS/RTPS 層偵測**（進階：Spicy 解析 RTPS）<br>③ 偵測 → 緊急停止聯動<br>④ **偵測率 / 誤報率驗證**<br>⑤ TB3 + Gazebo + 幾何巡邏（產生流量的測試台，最小夠用） | 這就是核心本體；深度與時間都投在這 |
| 🟧 **加分**（已完成，保留，**不再挖深**） | 應用層 HMAC envelope v3、行為 IDS D1–D6、威脅模型 | 是攻防打出來的好成果，但屬 **ROS 應用層、非 DDS 層**；報告裡以「defense-in-depth（應用層）」講一段即可 |
| 🟦 **砍 / 未來工作** | RL/TQC 自主巡邏、多環境泛化、LLM 模糊測試器、商業級 CVSS 包裝 | 與「DDS 攻防」無關 → 進未來工作（見 §4） |

> **為什麼應用層是「加分」不是「核心」**：核心被「專注 DDS」收斂到 **DDS 協定層（RTPS over UDP）**。HMAC 簽在 ROS 的 String 訊息上，是 DDS *之上* 的應用層 —— 保留它（真功夫），但深度投資回到 DDS 層。

## 2. 計劃「預期成果」6 項 vs 現況

| # | 預期成果 | 狀態 | 缺口 |
|---|---|------|------|
| 1 | 工廠模擬 + DDS **正常流量基準量測** | 🟡 | 模擬✓；缺正式「正常流量基準」數據 |
| 2 | 威脅模型 + **三類攻擊腳本** | 🟢 超額 | 有 N1–N24+，但按回合排，**需收斂成三類** |
| 3 | **Zeek** DDS 偵測 + 即時告警 | 🟡 偏薄 | 只抓「新節點」(偵察)；**偽造注入 / DoS 未偵測** |
| 4 | **Spicy** RTPS 解析器（進階） | 🔴 | 未做（核心的進階深度方向） |
| 5 | 告警 → **緊急停止聯動** | 🟡 | 應用層已通；**Zeek 端未接緊急停止** |
| 6 | **偵測率 / 誤報率驗證** | 🔴 | **完全未做**，最該補（攻防有效性的證據） |

## 3. 「挖深 DDS」要挖哪（核心的深度，對應教授指示）

1. **真實業界 / 規範**：OMG DDS-Security 規範、SROS2 三插件（認證 / 存取控制 / 加密）、為何工廠實務常停在 Permissive
2. **攻擊面**：RTPS 封包結構、哪些欄位可偽造、**加密前 vs 加密後 Zeek 各看得到什麼**
3. **偵測邏輯**：Zeek 從「連線層（新節點）」深入到 **RTPS submessage 層（DATA / HEARTBEAT / GAP）**，偵測 flood / inject pattern ← Spicy 的價值
4. **驗證**：攻擊 + 正常流量交替，算 TPR / FPR / 混淆矩陣（#6）

---

## 4. 未來可擴充方向（展現想得遠，不需現在花時間）

- **RL 自主巡邏（TQC）**：以強化學習取代幾何巡邏，提升動態避障；含多環境泛化訓練（held-out eval）
- **多環境泛化**：多張工廠地圖訓練 / 留存測試，量化導航泛化能力
- **LLM 模糊測試**：以 LLM 自動生成惡意 payload，擴大攻擊面覆蓋
- **跨主機 / 真實網段攻防**：從單機隔離擴展到跨主機 DDS domain
- **SROS2 Enforce 遷移**：DDS 層強制存取控制（目前 Permissive）
