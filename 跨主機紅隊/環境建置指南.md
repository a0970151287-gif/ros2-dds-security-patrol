# 跨主機 DDS 攻防環境 — 建置指南

> **目標**：把紅隊從「同機 localhost」搬到「真跨主機」，讓 DDS 流量真的走網路，**Zeek 才看得到偽造注入 / DoS 的封包**。
> **你的環境**：LAN = `192.168.100.0/24`（Windows 主機 Wi-Fi = `192.168.100.48`）；機器人機 = Windows 11 + WSL2（目前 NAT `172.30.x`）。

```
        LAN 192.168.100.0/24
   ┌──────────────┐        ┌──────────────┐
   │  機器人機(防) │◄──DDS──►│  攻擊機(攻)   │
   │  Gazebo+ROS  │ 7400+  │  ROS+紅隊腳本 │
   │  + Zeek 監聽  │        │              │
   └──────────────┘        └──────────────┘
     本機(WSL2)              第二台實體電腦
```

---

## 階段 1：機器人機（本機 WSL2）上 LAN —— mirrored 網路模式

WSL2 預設 NAT，LAN 上的攻擊機看不到它、multicast 也不通。Win11 的 mirrored 模式可解。

**1-1. 建立 `C:\Users\Jesse\.wslconfig`**（在 Windows 用記事本，內容）：
```ini
[wsl2]
networkingMode=mirrored

[experimental]
hostAddressLoopback=true
```

**1-2. 重啟 WSL**（Windows PowerShell）：
```powershell
wsl --shutdown
```
> ⚠️ 這會關閉所有 WSL 視窗（含目前這個對話的終端）。存好東西再做，重開 WSL 即可。

**1-3. 驗證**（重開 WSL 後）：
```bash
ip -4 addr show eth0 | grep inet
# 期望：inet 192.168.100.x   ← 上 LAN 了（不再是 172.30.x）
```

**1-4. 開 Windows 防火牆讓 DDS 進來**（PowerShell 系統管理員）：
```powershell
New-NetFirewallRule -DisplayName "ROS2 DDS UDP" -Direction Inbound `
  -Protocol UDP -LocalPort 7400-7600 -Action Allow
```
> mirrored 模式若仍收不到，補一條 Hyper-V 防火牆放行：
> `Set-NetFirewallHyperVVMSetting -Name '{40E0AC32-46A5-438A-A0B2-2B479E8F2E90}' -DefaultInboundAction Allow`

## 階段 2：攻擊機（第二台實體電腦）

**情況 A — 原生 Ubuntu（最省事，建議攻擊機用這個）**
```bash
# 裝 ROS2 Jazzy（或與機器人機相同版本）後：
git clone <你的 GitHub repo> ~/ros2_ws && cd ~/ros2_ws
colcon build --packages-select dds_security_monitor turtlebot3_dqn
```

**情況 B — 也是 Win11 + WSL2**：照「階段 1」同樣設 mirrored + 防火牆。

> 攻擊機**不需要** Gazebo，只要 ROS2 + 紅隊腳本（`紅隊測試/PoC腳本/`）。

---

## 階段 3：兩端共同的 ROS 環境變數

**兩台機器都要**（加進各自 `~/.bashrc`）：
```bash
export ROS_DOMAIN_ID=30          # 兩台一致，任選同一個值
unset ROS_LOCALHOST_ONLY         # 關鍵：不可設 1，否則只走本機
# 攻擊機若要驗證「未授權節點」場景，故意不 source SROS2 keystore
```

---

## 階段 4：驗證跨主機 DDS 通了（沒通就別往下做）

**4-1. multicast 雙向**（A 機 receive、B 機 send）：
```bash
# 機器人機： ros2 multicast receive
# 攻擊機：   ros2 multicast send
# 收得到 = multicast OK
```
**4-2. talker / listener 跨機**：
```bash
# 機器人機： ros2 run demo_nodes_cpp talker
# 攻擊機：   ros2 run demo_nodes_cpp listener   # 應印出 "I heard: Hello World"
```
任一步失敗 → 多半是防火牆或 mirrored 沒生效，回階段 1 檢查。

---

## 階段 5：Zeek 看得到跨主機 DDS 流量

在**機器人機**抓 LAN 介面（mirrored 後 eth0 即 LAN）：
```bash
cd ~/ros2_ws/Zeek監控
sudo zeek -i eth0 dds_monitor.zeek
# 另一終端跑 talker/紅隊；Zeek 應印 [DDS 節點警報]
# 進階驗證封包真的在線上：
sudo tcpdump -i eth0 -n 'udp portrange 7400-7600' | head
```
**這一步看到攻擊機的封包 = 整個跨主機核心成立**，接著才有意義做 #3（Zeek 偵測偽造注入/DoS）與 #6（偵測率驗證）。
